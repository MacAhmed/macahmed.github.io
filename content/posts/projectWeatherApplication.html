I recently completed a fairly simple Weather Application. The application displays the weather information for wherever you are located for the next 5 days. I am sure most of my code is very buggy and hacky, however it is the first actual application that i have developed so i am still excited regardless.

So, the basic HTML page is defined using Bootstrap. I feel that it is a bit of a overkill since i only really used it for the grid system. The icons i have used are another CSS library, called Weather Icons. I do not regret using this as its been extremely useful and it was quite helpful having a set of icons that went together.

The Javascript function that updates and basically renders the page is found in one file called app.js. There is one main function that provides most of the functionality. Using a $(document).ready to start the main code. I then use the geolocation api to get the currentPosition, prase it into the URL and then get the JSON returned and save it in a variable. I then run a loop that goes through the JSON array as well as the items on my page and updates them according to the data in JSON.

I created three helper functions getAPIData(restURL), dayOfWeekAsString(dayIndex), provideIcon(iconID).

The getAPIData(restURL) feels like to me that its a waste and there has to exist some function that potentially perform this task better. I have imported jQuery but am not sure if a function like such exists. 

function getAPIData(restURL){
    var xhttp = new XMLHttpRequest();
    xhttp.open("GET", restURL , false);
    xhttp.send();
    var response = JSON.parse(xhttp.responseText);

    return response;
}
The above technique of doing this is something that i think is not strictly correct. This is because, when used, i am getting a error message in the console like so : Synchronous XMLHttpRequest on the main thread is deprecated because of its detrimental effects to the end user's experience.
